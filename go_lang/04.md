# ‚úÖ **Difference Between Methods and Functions in Go**

In Go, **every method is a function**, but not every function is a method.

The difference is:

# üîµ **1. Functions**

A **function** belongs to *no type*.
It is independent and can be called directly.

```go
func add(a int, b int) int {
    return a + b
}
```

You call it like:

```go
result := add(10, 20)
```

---

# üî¥ **2. Methods**

A **method** is a function that **belongs to a type** (struct or alias).
It has a **receiver**, which is like a lightweight version of ‚Äúthis‚Äù or ‚Äúself‚Äù.

Syntax:

```go
func (receiver Type) methodName() {
    ...
}
```

Example:

```go
type User struct {
    Name string
    Age  int
}

func (u User) Greet() {
    fmt.Println("Hello,", u.Name)
}
```

You call it using a value:

```go
user := User{"Sandeep", 21}
user.Greet()
```

# üî• **Core Difference Table**

| Feature            | Function         | Method                      |
| ------------------ | ---------------- | --------------------------- |
| Receiver?          | ‚ùå No             | ‚úî Yes                       |
| Belongs to a type? | ‚ùå No             | ‚úî Yes                       |
| Syntax             | `func name(...)` | `func (t Type) name(...)`   |
| Called on          | Directly         | Type value or pointer       |
| Use case           | Utility logic    | Behavior of structs/objects |

# üîµ **Detailed Real Example (Simple)**

## **Function example**

```go
func area(width, height int) int {
    return width * height
}
```

## **Method example**

```go
type Rectangle struct {
    Width  int
    Height int
}

func (r Rectangle) Area() int {
    return r.Width * r.Height
}
```

### Function call:

```go
println(area(10, 20))
```

### Method call:

```go
rect := Rectangle{10, 20}
println(rect.Area())
```

---

# üî¥ **Value Receiver vs Pointer Receiver (Important!)**

Go allows both:

### ‚úî **Value receiver**

Method gets a copy of the struct.

```go
func (u User) Greet() {
    fmt.Println("Hi,", u.Name)
}
```

### ‚úî **Pointer receiver**

Method can **modify** the original struct.

```go
func (u *User) UpdateAge(newAge int) {
    u.Age = newAge
}
```

# üü¢ **Final Example: Methods vs Functions Together**

```go
type BankAccount struct {
    Balance float64
}

// Method: modifies the struct
func (a *BankAccount) Deposit(amount float64) {
    a.Balance += amount
}

// Function: utility not tied to a type
func convertToUSD(inr float64) float64 {
    return inr / 83.0
}

func main() {
    acc := BankAccount{Balance: 1000}
    acc.Deposit(500)

    fmt.Println("Balance:", acc.Balance)
    fmt.Println("In USD:", convertToUSD(acc.Balance))
}
```

# üéØ **Summary (Interview Ready)**

### ‚úî A **function** is independent logic.

### ‚úî A **method** is a function with a **receiver**, meaning it is tied to a type.

### ‚úî Methods define **behavior** of structs.

### ‚úî Functions define **general utility logic**.
